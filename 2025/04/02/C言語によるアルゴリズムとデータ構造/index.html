1.アルゴリズムの概念

   アルゴリズム（algorithm） の基本概念は、問題を解決するための具体的な手順を示すものです。 一般的に、アルゴリズムは入力データを受け取り、特定のルールに従って処理し、出力結果を生成します。 この過程は、明確で順序立てられたものでなければなりません。

2.簡単なアルゴリズムの例

[例１] $12345$

原始的な方法：

ステップ①：まず1×2を解き、結果2を得る。

ステップ②： ステップ1で得た積2に3を掛けて、結果6を得る。

ステップ③：6に4を掛けて24を得る。

ステップ④： 24に5を掛けて120を得る。

このようなアルゴリズムは正しいが、あまりに面倒だ

改良した方法：

S1：t=1 とする

S2：i=2 とする

S3：t×i とする、積は変数tに置かれたままで、t×i→t と表現できる。

S4：iの値 を ＋1、つまり i＋1→i とする。

S5: i ≤ 5 の場合、ステップS3とそれに続くS4とS5の再実行に戻る。

100を計算する場合は、S5: if i ≤ 5 を i ≤ 100 に変更する。

もし1×3×5×7×9×11を求めるのであれば、このアルゴリズムもほとんど変更する必要はない：

S1: 1→t

S2: 3→i

S3: t×i→t

S4: i+2→t

S5: i≤11 なら S3 に戻り、そうでなければ終了。

このアルゴリズムは正しいだけでなく、コンピュータは高速計算が可能な自動機械であり、ループの実装も簡単であるため、コンピュータにとってはより優れたアルゴリズムである。

C言語で書く場合

[例2] 50人の学生がいて、成績が80点以上の学生を印刷する必要がある。

n が生徒番号、ni が最初の生徒番号、g が生徒の成績、gi が最初の生徒の成績、とすると、アルゴリズムは以下のように表現できる：

S1: 1→i

S2: gi≧80 ならば、ni と gi を印刷し、そうでなければ印刷しない

S3: i+1→i

S4: i≦50 ならば、S2 に戻り、そうでなければ終了。

C言語で書く場合

[例3] 3以上の正の整数が素数かどうかを判断する

アルゴリズムは以下のように表される：

S1：n の値を入力する

S2：i = 2 とする

S3：n を i で割り、余り r を求める

S4：もし r = 0 ならば、n は i で割り切れることを意味するので、n は「素数ではない」と出力して、アルゴリズムを終了する。そうでなければ S5 に進む。

S5：i に 1 を加える（i = i + 1）

S6：もし i ≤ n - 1 ならば、S3 に戻る。そうでなければ、n は「素数である」と出力して、アルゴリズムを終了する。

3.アルゴリズムの特徴

・有限性：アルゴリズムは、無限ではなく、有限の手順で構成されていなければならない。

・確定性：アルゴリズムの各ステップは、あいまいではなく、明確でなければならない。

・0個以上の入力がある。

・1個以上の出力がある。

・有効性：アルゴリズムの各ステップは、現実的に実行可能であり、明確な結果が得られなければならない。

  プログラマーにとって、アルゴリズムを設計できること、そしてそのアルゴリズムに基づいてプログラムを書くことが重要である。
